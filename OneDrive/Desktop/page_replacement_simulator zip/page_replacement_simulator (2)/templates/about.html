Efficient page replacement algorithms are crucial for managing memory in operating systems. They determine which pages to swap out when new pages need to be loaded into memory.

FIFO (First-In-First-Out)

This algorithm replaces the oldest page in memory. It maintains a queue of pages, and when a page needs to be replaced, the page that has been in memory the longest is removed.

Example: If pages 1, 2, and 3 are in memory and page 1 is the oldest, when page 4 is requested, page 1 is replaced.

Page Request Memory State Page Replaced Page Faults
1 [1] - 1
2 [1, 2] - 2
3 [1, 2, 3] - 3
4 [2, 3, 4] 1 4

LRU (Least Recently Used)

This algorithm replaces the page that has not been used for the longest period of time. It keeps track of page usage over time and removes the least recently accessed page.

Example: If pages 1, 2, and 3 are in memory and page 2 is the least recently used, when page 4 is requested, page 2 is replaced.

Page Request Memory State Page Replaced Page Faults
1 [1] - 1
2 [1, 2] - 2
3 [1, 2, 3] - 3
4 [1, 3, 4] 2 4

Optimal

This algorithm replaces the page that will not be used for the longest period in the future. While it provides the best performance, it is not practical for implementation since it requires future knowledge of page requests.

Example: If pages 1, 2, and 3 are in memory and the next requests are for pages 1, 4, and 2, the optimal choice would be to replace page 3 since it will not be used again.

Page Request Memory State Page Replaced Page Faults
1 [1] - 1
2 [1, 2] - 2
3 [1, 2, 3] - 3
4 [1, 2, 4] 3 4
